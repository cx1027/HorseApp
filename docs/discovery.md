# Discovery

## Confirmed assumptions
- Use the provided Figma file as a visual reference only for palette, typography, spacing, component style, and UI tone.
- Do not copy old app architecture, wording, labels, or business logic.
- Phase 1 payments: no payment gateway.
- Phase 1 notifications: in-app only.

## Core users
- Admin / Management
- Professional staff / horse managers
- Horse owners
- Members / investors

## Business objective
Deliver a clean PWA-first web app for viewing and sharing horse profiles, ownership/share summaries, updates, documents, and investor-facing reporting.

## Key nouns
- horse profile
- ownership / share
- health / weight / feeding
- race and commercial records
- insurance
- investor report
- document
- notification

## Success criteria for Phase 1
- A logged-in owner or investor can open assigned horses, review ownership summary, and read a report within a few taps.
- A staff user can update horse records and upload documents without leaving the app.
- Admins can manage users and horses without developer help.
- The app installs on mobile through PWA add-to-home-screen behavior.

## Out of scope for Phase 1
- payments/subscriptions
- native App Store/Play Store binaries
- AI-generated insights
- third-party race-entry integrations
- in-app email delivery
